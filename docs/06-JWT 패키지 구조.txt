2. property 구조
    application-jwt.yml 파일에 필요한 프로퍼티만 설정을 하고
    application.yml에서 해당 파일을 import 하는 방식을 사용한다.

    [application-jwt.yml]
    jwt:
      secret: ${JWT_SECRET}
      access-token-expiration: 1800000
      refresh-token-expiration: 1209600000
      issuer: kerusia

    [application.yml]
    spring:
      config:
        import: classpath:application-jwt.yml

    [Class]
    @ConfigurationProperties(prefix = "jwt")
    public record JwtProperties(
        String secret,
        long accessTokenExpiration,
        long refreshTokenExpiration,
        String issuer) { }

    [Annotation]
    @ConfigurationPropertiesScan << 추가
    @SpringBootApplication
    public static void main(String[] args) { ... }

Builder는 단지 "JWT를 만들기 위한 설계도"를 만드는 것이다. 실제 Header, Payload, Signature 생성은 마지막 compact()를 호출하는 순간 한 번에 수행

builder()

↓

subject()

↓

claim()

↓

expiration()

↓

signWith()

↓

compact()   ← 여기서 Header/Payload/Signature 생성

[Jwts.builder()]
issuer(): 발급자(iss)
subject(): 주체(회원번호, 아이디, UUID)
audience(): 토큰 사용자(aud)
expiration(): 유효기간(exp)
issuedAt(): 생성일시(iat)
notBefore(): 유효기간 시작일시(nbf)
id(): JWT ID
claim(): Payload
signWith(): HMACSHA512 해싱으로 Signature 생성
compact(): String 토큰 생성

[Jwts.parser()]
Claims claims =
    Jwts.parser()
        .verifyWith(secretKey)
        .build()
        .parseSignedClaims(token)
        .getPayload();
